
1. **Handler 缓存**：修改 `infer_processors`（如 3-way 的 pre-norm）后，需删除对应 `handler_*.pkl` 才会重新构建数据。
2. **随机种子**：控制 PyTorch/NumPy 初始化、Dropout、日度 batch 打乱顺序等；多 seed 可评估稳定性。
3. **qlib 数据**：默认 `provider_uri: ~/.qlib/qlib_data/cn_data`，需事先准备数据。

```bash
cd /root/WaveFormer/qlib/examples/benchmarks/Waveformer
```

## 1. 正常训练 + 回测

使用默认配置 `workflow_config_waveformer_Alpha158_lowturn.yaml`（默认 `use_pre_norm_wavelet: true` 且 `use_wavelet_denoise: false`，即单次运行为 **pre_norm** 去噪）：

```bash
python main_waveformer.py
```

指定配置与日志目录：

```bash
python main_waveformer.py --config ./workflow_config_waveformer_Alpha158_lowturn.yaml --log_dir logs
```

若 `model/` 下已存在与当前 `save_prefix`、分组、seed 一致的 checkpoint，会**跳过训练**直接加载并跑信号与回测。

单次运行（非 `--ablation`）时权重命名规则（`save_prefix` 默认为 YAML 中的 `csi300`）：

- `use_pre_norm_wavelet: true`（默认）→ `model/csi300_pre_norm_{seed}.pkl`（与 3-way 消融第二组一致；handler 缓存为 `handler_*_*_pre_norm.pkl`）
- `use_pre_norm_wavelet: false` 且 `use_wavelet_denoise: false` → `model/csi300_baseline_{seed}.pkl`
- `use_pre_norm_wavelet: false` 且 `use_wavelet_denoise: true` → `model/csi300_after_norm_{seed}.pkl`（与消融 after_norm 组一致）

若同时开启 `use_pre_norm_wavelet` 与 `use_wavelet_denoise`，脚本会只用 pre_norm 数据并关闭模型内去噪（避免双重去噪）。

## 4. 小波去噪相关命令行参数

在非消融单次运行时，这些参数会写入 `config["task"]["model"]["kwargs"]`；消融中 **after_norm** 组同样会用到。

| 参数 | 说明 |
|------|------|
| `--wavelet` | 默认 `haar` |
| `--denoise_level` | 默认 `1` |
| `--threshold_method` | `bayes` \| `visu`，默认 `bayes` |
| `--threshold_mode` | `soft` \| `hard` \| `semisoft`，默认 `soft` |
| `--threshold_scale` | 默认 `0.3` |
| `--denoise_blend` | 默认 `0.25` |
| `--no_denoise_finest_only` | 关闭「仅最细层」阈值 |
| `--no_level_dependent_scale` | 关闭层级相关缩放 |
| `--use_edge_pad` / `--no_edge_pad` | 边缘填充（默认开启） |
| `--use_boundary_smooth` | 边界平滑（默认关闭） |
| `--boundary_smooth_win` | 边界平滑窗口，默认 `1` |

示例：

```bash
python main_waveformer.py \
  --wavelet haar \
  --denoise_level 1 \
  --threshold_method bayes \
  --threshold_mode soft \
  --threshold_scale 0.3 \
  --denoise_blend 0.25 \
  --use_boundary_smooth \
  --boundary_smooth_win 1
```

## 5. 配置与回测区间说明

- **训练/验证/测试数据划分、标签、step_len** 等：见 `workflow_config_waveformer_Alpha158_lowturn.yaml` 中 `task.dataset`。
- **回测起止时间、策略（TopK、n_drop、hold_thresh 等）**：同一 YAML 中的 `port_analysis_config`（`backtest.start_time` / `end_time` 等）。

当前 `workflow_config_waveformer_Alpha158_lowturn.yaml` 中划分（在 `end_time=2026-03-13` 前提下尽量多用数据训练）：

- **训练**：`2008-01-01`～`2019-12-31`（约 12 年）
- **验证**：`2020-01-01`～`2021-12-31`（约 2 年）
- **测试**：`2022-01-01`～`2026-03-13`（与回测区间一致）
- **`fit_end_time`**：`2019-12-31`，与训练段结束对齐，供 `RobustZScoreNorm` 拟合

更新数据后请运行 `python check_qlib_data_range.py` 核对末日，并相应改 `end_time` / `test` / `backtest.end_time`。

修改划分后，首次运行会生成新的 handler 缓存（如 `handler_20080101_20260313.pkl`）；旧日期的 `handler_*.pkl` 可手动删除以免混淆。**划分变更后建议重新训练**，旧 checkpoint 与新区间不匹配。
